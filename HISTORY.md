### Carpentry Concierge deploy — argv bootstrap gotcha (2026-07-14)
First admin bootstrap failed to log in because the PIN-hash generator was run as a file (`node hash.js PIN`) after the README's `-e` one-liner (`node -e '…' PIN`). In `-e` mode the PIN is `process.argv[1]`; in file mode `argv[1]` is the script path and the PIN is `argv[2]`. The script kept `argv[1]`, so it hashed the **file path**, not the PIN — a valid PBKDF2 hash of a Windows path. Symptom: correct PIN returns 401 while auth machinery is provably fine. Fix: regenerate with `argv[2]` (file mode) and `UPDATE members SET pin_hash=… WHERE phone=…`. Lesson for any future bootstrap doc: pin the argv index to the invocation mode.
